♻️ GreenLens 프로젝트 정리

AI 기반 분리수거 인식 앱 — 서버 & 안드로이드 클라이언트 전체 구조 문서

Spring Boot 3.4.4 Android (minSdk 24) YOLOv5 Gemini 2.0 Flash MySQL (Aiven) JWT 인증 Retrofit2

① 프로젝트 개요

GreenLens는 카메라로 재활용 물품을 촬영하면 AI가 자동으로 분류하고, 포인트를 지급하는 환경 앱입니다.
백엔드는 Spring Boot로 구성되고, 프론트는 Android(Java)로 작성되었습니다.

집계 기준

recycle_log 테이블의 기록 횟수로 순위를 매깁니다. AI 분석 1회 = 로그 1개 생성. 

-- RankingService.java 에서 실행되는 쿼리
SELECT r.user.id, r.user.username, COUNT(r)
FROM RecycleLog r
WHERE r.user IS NOT NULL
GROUP BY r.user.id, r.user.username
ORDER BY COUNT(r) DESC

⑧ 인증 / 토큰 관리

서버 측

  • 로그인 성공 시 JWT 토큰 발급

  • @PreAuthorize("isAuthenticated()")으로 보호

앱 측 (UserManager.java)

  • 토큰을 SharedPreferences에 저장
  • isTokenExpired()로 JWT exp 클레임 직접 파싱
  • 만료 시 세션 초기화 후 로그인 화면으로
  • 모든 API 헤더에 Bearer {token} 자동 첨부

⑩ 수정된 버그 (이번 세션)

버그 1 — 페트병 인식 → 알수없음 페이지
YOLO/Gemini가 "bottle", "유리병", "알루미늄캔" 등 비표준 값을 반환할 때 매핑 실패.
→ 서버에 normalizeCategory() 추가, Gemini 프롬프트에 허용 목록 명시, 클라이언트 매핑 확장.
버그 2 — 랭킹 시스템 동작 안함
RankingViewModel이 UserManager 매개변수 생성자만 가져 ViewModelProvider 기본 팩토리로 인스턴스 생성 불가 → 크래시.
→ ViewModel을 AndroidViewModel로 변경, Application에서 UserManager 획득.
버그 3 — 포인트 창 이후 결과 늦게 나옴
분석 API가 이미 category/disposal_method를 응답에 포함하는데, 불필요하게 GET /recycle/result/{id}를 한 번 더 호출해 추가 지연 발생.
→ AnalyzeResponse에서 직접 결과 사용, 다이얼로그 확인 즉시 showResult() 호출.
추가 개선
· remaining_reward_count 계산 오류 수정 (지급 후 차감 반영)
· bottomSheet.getDialog().setOnDismissListener() NPE 위험 → 콜백 인터페이스 패턴으로 교체
· setupResultView 내 죽은 코드 type = "plastic" 제거

RankingService.getLeaderboard()

@Transactional(readOnly = true)
public LeaderboardResponse getLeaderboard() {
    List<Object[]> rawData = recycleLogRepository.countRecycleLogsByUser();
    List<UserRankingDto> rankings = rawData.stream()
        .map(data -> new UserRankingDto(0, (Long)data[0], (String)data[1], (Long)data[2]))
        .collect(Collectors.toList());

    for (int i = 0; i < rankings.size(); i++) rankings.get(i).setRank(i + 1);
    return new LeaderboardResponse(ZonedDateTime.now(),
        recycleLogRepository.countDistinctUsers(),
        recycleLogRepository.countTotalRecycles(), rankings);
}

AnalysisResultResponse.mapCategoryToType() (수정 후)

public static String mapCategoryToType(String category) {
    switch (category.toLowerCase().trim()) {
        case "페트병": case "pet": case "bottle": case "pet_bottle":
            return "pet";
        case "유리": case "glass": case "유리병": case "glass_bottle":
            return "glass";
        case "캔": case "metal": case "알루미늄캔": case "철캔":
            return "metal";
        // ... 전체 매핑 생략
        default: return "plastic";
    }
}
